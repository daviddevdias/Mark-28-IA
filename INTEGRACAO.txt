COMO INTEGRAR — PASSO A PASSO
------------------------------

1. COPIE todos os .py para as pastas correspondentes
   (event_bus.py → engine/, memory_rag.py → storage/, etc.)

2. ATUALIZE main.py — adicione no início de `engine()`:

    from engine.event_bus import bus
    from engine.state_manager import state
    from engine.watchdog import watchdog, registrar_modulos_padrao
    from storage.memory_rag import rag

    # registrar loop no bus e watchdog
    bus.registrar_loop(loop)
    registrar_modulos_padrao()
    watchdog.iniciar()
    rag.purgar_antigos(dias=30)

3. ATUALIZE core.py — em `processar_comando()`:

    # Antes de router.responder():
    from storage.memory_rag import rag
    from engine.event_bus import bus, VOZ_RECEBIDA, IA_RESPOSTA

    await bus.publicar_async(VOZ_RECEBIDA, {"comando": comando})

    # Adicionar contexto RAG ao prompt:
    ctx_rag = rag.contexto_para_prompt(comando)
    memoria_completa = contexto() + ("\n" + ctx_rag if ctx_rag else "")

    # Após obter resposta:
    rag.registrar_interacao(comando, resposta)
    await bus.publicar_async(IA_RESPOSTA, {"resposta": resposta})

4. PARA USAR o agente em tarefas complexas (tools_mapper.py):

    # Em gerenciador_plano():
    from engine.agent_cycle import executar_tarefa_complexa
    return executar_no_loop_atual(executar_tarefa_complexa(objetivo, contexto))

5. PARA USAR busca multi-fonte (tools_mapper.py):

    # Em gerenciador_web():
    from tasks.smart_search import buscar_sync
    resultado = buscar_sync(pesquisa)

6. PARA USAR TTS com segmentação (core.py):

    from engine.tts_pipeline import falar_com_segmentacao
    # Substituir `await falar(resposta)` por:
    await falar_com_segmentacao(resposta, falar)

7. PARA USAR seletor de modelos (controller.py):

    from engine.model_selector import escolher_modelo
    modelo_ideal = escolher_modelo({
        "comando": pergunta,
        "imagem": imagem,
        "historico_len": len(historico),
    })


COMPATIBILIDADE
---------------
- Nenhum módulo existente foi modificado
- Todos os novos módulos têm imports opcionais (try/except)
- Podem ser ativados incrementalmente, um por vez
- Fallback automático para comportamento anterior em caso de erro


BENEFÍCIOS ESPERADOS
---------------------
1. Event Bus       → módulos desacoplados, fácil adicionar features
2. TTS Pipeline    → resposta percebida ~2x mais rápida
3. Multi-Modelo    → modelos leves para comandos simples (economia de RAM)
4. Memória RAG     → contexto relevante automático, respostas melhores
5. Agente Ciclo    → tarefas multi-etapa sem intervenção manual
6. Watchdog        → sistema se recupera sozinho de falhas
7. Tool Interface  → padrão único, fácil testar e expandir
8. Smart Search    → fallback Wikipedia + Playwright, mais confiável
9. Observability   → saber exatamente onde o sistema está lento
10. State Manager  → estado consistente, sem variáveis globais espalhadas
